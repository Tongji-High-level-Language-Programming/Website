# BitLocker加密功能科普与使用指南
BitLocker 是 Windows 内置的磁盘加密功能，用于保护存储在硬盘上的数据。
省流：不想看前面科普的，可以直接看第4部分的最佳实践和第5部分的恢复模式操作指南
## 1.BitLocker 是什么？有什么用？
BitLocker 是微软自 Windows Vista 起为专业版及以上系统（如 Win10/11 专业版、企业版、教育版）提供的全磁盘/分区加密功能。
主要作用：
●防范物理窃取：设备丢失或被盗后，他人无法将硬盘拆下挂载到其他电脑上读取数据。
●防范离线攻击：加密整个卷（包括系统文件、休眠文件等），可抵御通过PE启动盘等方式，绕过计算机登录密码进行的数据拷贝尝试。
●增强启动完整性：配合主板上的 TPM (可信平台模块) 芯片，可验证系统启动过程的完整性，防止启动文件被篡改。
●注意：Windows 家庭版不支持 BitLocker，但部分设备可能提供功能受限的“设备加密”功能。
## 2.BitLocker的使用
●透明加解密：Windows 10及以上的Windows操作系统中，Bitlocker通常默认启用。当用户正常登录 Windows 后，系统会自动使用受保护的密钥在后台完成数据的解密与加密，用户通常无需操作。
●启动验证：现代品牌机通常都搭载了TPM（可信平台模块），系统启动时会验证 BIOS、引导文件等是否被篡改。验证通过TPM才会释放密钥，允许系统启动；否则将进入恢复模式，要求输入恢复密钥。
## 3.使用 BitLocker 的主要风险
BitLocker的风险主要集中在密钥管理和操作失误上。
●恢复密钥丢失 = 数据永久丢失 BitLocker 的设计原则就是“没有密钥就无法解密”。如果忘记了密码，且丢失了备份的恢复密钥，数据基本无法通过常规手段恢复。
●加密/解密过程意外中断 在加密或解密过程中，若遭遇断电、系统崩溃或强制关机，可能导致磁盘数据结构损坏，造成数据丢失或无法访问。
●系统或硬件变更触发恢复模式 更换主板、升级 BIOS/UEFI 固件、修改启动顺序、TPM 重置等操作，都可能使 BitLocker 判定系统环境不安全，从而进入恢复模式，要求输入恢复密钥。在某些条件下，恢复模式可能被误触发。
●对性能的影响 在机械硬盘上，加密/解密会占用一定的 CPU 资源。在部分配置下，对 SSD 的随机读写性能可能有轻微影响，但多数用户在日常使用中难以感知。
## 4.最佳实践
●由于Bitlocker采用XTS-AES-128加密，如果进入恢复模式，在没有恢复密钥的情况下，解密数据的可能性几乎为0，请务必保管好恢复密钥。
●通常而言，品牌机在初次启动时，OOBE会引导你创建Microsoft账户，如果设备启动了Bitlocker加密，通常而言恢复密钥会被自动备份到Microsoft账户。

https://account.microsoft.com/devices/recoverykey

## 5.恢复模式
当系统启动时，若检测到不安全因素，Bitlocker会进入恢复模式，要求输入48位恢复密钥。这种情况下，由于Bitlocker采用XTS-AES-128加密，没有恢复密钥，破解的可能性几乎为0
提示：恢复密钥 ID（恢复界面上提示的前 8 位字符）可以辅助在多个密钥中快速找到正确的那一个。
●Microsoft账户：在另一台设备上访问网址
https://account.microsoft.com/devices/recoverykey
登录您的Microsoft账户，您可以在这里检阅您备份到Microsoft账户的所有恢复密钥。
●打印的纸质备份：如果您曾创建过纸质备份，您可以找到它并根据它输入恢复密钥。
●U 盘备份：在曾用于备份恢复密钥的U盘中查找恢复密钥文件，若您没有自定义名称，默认名称应当为BitLocker Recovery Key {一串字符}.txt。
●本地/网络位置：如果您曾手动将密钥文件保存在其他硬盘或网络共享中，请前往对应位置查找。
